# OctoAcme Personas

This document defines typical roles and responsibilities used in OctoAcme project docs and exercises.

---

## Developers

### Role Summary
Developers design, build, test, and deliver software components. They collaborate with product and project leads to implement features that meet acceptance criteria and quality standards.

### Responsibilities
- Implement features and fixes to meet acceptance criteria
- Write and maintain tests and documentation
- Participate in design and code reviews
- Assist in estimating and planning work
- Help identify technical risks and propose mitigations

### Goals
- Deliver reliable, maintainable code
- Reduce cycle time from idea to production
- Maintain high test coverage and observability

### Typical Communication
- Daily standups and sprint planning
- PR descriptions and code review comments
- Technical design docs when needed

### Key Interactions
- **With Product Manager**: Weekly syncs on acceptance criteria, scope clarifications
- **With QA/Testers**: Continuous feedback on bugs, test environment support
- **With UX Designer**: Design validation, implementation feasibility discussions
- **With Technical Writer**: Technical documentation review and API clarifications

---

## Product Managers

### Role Summary
Product Managers define what should be built to deliver customer and business value. They own the product vision, prioritize the backlog, and measure outcomes.

### Responsibilities
- Define problem statements and success metrics
- Prioritize the roadmap and backlog
- Collaborate with stakeholders and engineering on trade-offs
- Validate solutions through user research and metrics

### Goals
- Maximize customer value and impact
- Make clear, data-driven prioritization decisions
- Ensure product-market fit and usability

### Typical Communication
- Weekly alignment with PM and engineering leads
- Roadmap updates and stakeholder briefings
- Acceptance criteria and feature specs

### Key Interactions
- **With Developers**: Daily standups, acceptance criteria clarification, technical trade-off discussions
- **With Project Manager**: Weekly 1:1 on timeline, capacity, and risks
- **With Data Analyst**: Metrics definition, impact analysis, trend interpretation
- **With UX Designer**: Design prioritization, user research synthesis, design reviews

---

## Project Managers

### Role Summary
Project Managers coordinate delivery activities, manage schedules, risks, and communications. They enable the team to deliver on commitments efficiently.

### Responsibilities
- Create and maintain project plans and timelines
- Manage risks, dependencies, and resource constraints
- Facilitate meetings (kickoff, planning, retrospectives)
- Ensure consistent project documentation and status reporting
- Coordinate cross-team and stakeholder communication

### Goals
- Deliver projects on time and within scope
- Minimize unplanned work and escalations
- Maintain transparency and alignment across stakeholders

### Typical Communication
- Weekly status updates and stakeholder reports
- Risk registers and decision logs
- Coordination via project boards and meeting facilitation

### Key Interactions
- **With Product Manager**: Weekly alignment, priority shifts, resource conflicts
- **With Developers**: Risk assessment, timeline feasibility, blocker resolution
- **With QA/Testers**: Test timeline coordination, release readiness
- **With all roles**: Weekly sync facilitation, stakeholder communication

---

## QA / Testing

### Role Summary
QA professionals ensure quality through testing strategy, execution, and validation. They work closely with developers and product managers to confirm features meet acceptance criteria and quality standards.

### Responsibilities
- Define test strategy and acceptance criteria validation approach
- Execute manual and automated tests
- Report bugs with clear reproduction steps
- Validate fixes and sign-off on features
- Contribute to quality metrics and reporting

### Goals
- Ensure features meet acceptance criteria before release
- Identify issues early in development
- Maintain quality standards and reduce production incidents

### Typical Communication
- Weekly test planning with developers
- Bug reports and test case documentation
- QA sign-off in release checklists

### Key Interactions
- **With Developers**: Bug triage, test environment setup, reproduction support
- **With Product Manager**: Acceptance criteria validation, edge case discussion
- **With Project Manager**: Release timeline coordination, QA blockers
- **With Technical Writer**: Test documentation, user guide validation

---

## UX Designer

### Role Summary
UX Designers drive user research, create design specs, and validate usability through prototyping. They ensure features are intuitive and meet user needs.

### Responsibilities
- Conduct user research and validate user needs
- Create wireframes, prototypes, and design specs
- Lead design reviews and iterate based on feedback
- Validate implementation matches design intent
- Support post-release UX validation

### Goals
- Ensure solutions are user-centered and intuitive
- Reduce usability issues and user confusion
- Align product with customer expectations

### Typical Communication
- Design kickoffs with product and engineering
- Design review meetings and feedback cycles
- Implementation validation and post-release checks

### Key Interactions
- **With Product Manager**: Research synthesis, design prioritization, feature scope
- **With Developers**: Technical feasibility, implementation questions, design validation
- **With Project Manager**: Design timeline, dependency communication
- **With Data Analyst**: User behavior analysis, post-release UX metrics

---

## Data Analyst

### Role Summary
Data Analysts define success metrics, track progress, and provide insights to inform product decisions. They work with product and project leads to measure impact and drive continuous improvement.

### Responsibilities
- Define and track key performance indicators (KPIs)
- Analyze results and identify trends
- Provide actionable insights for roadmap and prioritization
- Establish baselines and measure feature impact
- Report metrics to stakeholders

### Goals
- Enable data-informed decision-making
- Demonstrate product value and ROI
- Identify optimization opportunities

### Typical Communication
- Planning meetings for metrics definition
- Weekly or milestone-based metric reviews
- Post-release impact reporting

### Key Interactions
- **With Product Manager**: Metrics definition, trend analysis, business impact
- **With Project Manager**: Milestone tracking, success criteria baselines
- **With Developers**: Instrumentation requirements, data quality questions
- **With stakeholders**: Metric reporting, impact communication

---

## Technical Writer

### Role Summary
Technical Writers ensure all features, APIs, and processes are clearly documented for users and developers. They work across teams to gather information and deliver comprehensive documentation.

### Responsibilities
- Create and maintain user guides, API documentation, and release notes
- Gather technical information from developers and product managers
- Ensure documentation accuracy and clarity
- Validate documentation against implemented features
- Support onboarding and knowledge transfer

### Goals
- Reduce support tickets through clear documentation
- Enable self-service learning for users and developers
- Maintain up-to-date, discoverable documentation

### Typical Communication
- Planning meetings for documentation scope
- Weekly documentation reviews with developers
- Release coordination for documentation delivery

### Key Interactions
- **With Developers**: Technical details, API specs, implementation questions
- **With Product Manager**: Feature context, use cases, user-facing messaging
- **With Project Manager**: Release timeline, documentation deadlines
- **With QA/Testers**: Edge cases, known issues, test documentation
